Notas sobre relaciones

  • «include»: el caso de uso incluido siempre se ejecuta como parte del principal (ej. crear cualquier pedido siempre imprime ticket).
  • «extend»: el caso de uso extensor se ejecuta solo bajo ciertas condiciones (ej. gestión económica solo si hay diferencia de importe o reembolso pendiente).
  • RehacarUltimoPedido extiende RealizarPedidoWeb porque es una variante opcional que pre-rellena el carrito con líneas anteriores.
  • Canal WhatsApp y la mayor parte del Panel Admin están planificados y parcialmente implementados.
